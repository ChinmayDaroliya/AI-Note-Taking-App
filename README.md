# AI Note-Taking App (PErN Stack POC)

> **Live demo:** https://ai-note-taking-app-ten.vercel.app/
>
> This project is deployed and available for exploration at the link above.
>
>A full-stack note-taking application built with **Next.js (App Router)**, **TypeScript**, **Supabase Auth**, **Prisma + PostgreSQL**, and **AI-powered note enhancements** via the **GROQ (OpenAI-compatible)** API.

---

## ✅ Core Features

- **Authentication**
  - Email/password signup & login via Supabase Auth
  - Protected routes (requires login to access notes)
  - Simple user profile stored in `profiles` table

- **Notes Management**
  - Create, view, edit, and delete notes
  - Search notes by title
  - Notes stored in PostgreSQL via Prisma (`notes` table)
  - Tags support (auto-generated by AI)

- **AI Features** (powered by GROQ / OpenAI API)
  - **Summarize** long notes
  - **Improve** language (grammar/clarity)
  - **Auto-generate tags** based on note content

- **UI / UX**
  - Clean responsive UI with **shadcn/ui** + Tailwind CSS
  - Dark/light theme toggle (persisted in localStorage)
  - Rich animated interactions using Framer Motion

---

## 🧰 Tech Stack

- **Frontend:** Next.js 14 (App Router), TypeScript, React
- **Styling:** Tailwind CSS, shadcn/ui
- **Backend:** Next.js API routes (Hono-style in App Router)
- **Database:** PostgreSQL + Prisma ORM
- **Auth:** Supabase Auth (email/password)
- **AI:** GROQ (OpenAI-compatible) via `openai` package
- **State/data fetching:** React Query (@tanstack/react-query)

## 🚀 Getting Started (Local)

### 1) Clone

```bash
git clone <https://github.com/ChinmayDaroliya/AI-Note-Taking-App.git>
cd peacock-pern-assignment
```

### 2) Install dependencies

```bash
npm install
```

### 3) Create environment variables

Create a `.env.local` file in the project root with the following values:

```env
# PostgreSQL (required for Prisma)
DATABASE_URL=postgresql://USER:PASSWORD@HOST:PORT/DATABASE

# Supabase Auth (required for login)
NEXT_PUBLIC_SUPABASE_URL=https://<your-supabase-project>.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=<your-public-anon-key>

# AI (GROQ / OpenAI-compatible)
GROQ_API_KEY=<your-groq-or-openai-api-key>
```

> 🔥 Tip: If you don’t have a Supabase project yet, you can sign up at https://supabase.com and create a new project.

### 4) Run Prisma migrations (create tables)

```bash
npx prisma migrate dev --name init
```

### 5) Start development server

```bash
npm run dev
```

Open http://localhost:3000 in your browser.

## 🧩 Project Structure (Key Parts)

- `app/` – Next.js App Router pages and API routes
  - `app/page.tsx` – Notes dashboard (list + search)
  - `app/auth/page.tsx` – Authentication (sign in / sign up)
  - `app/note/[id]/page.tsx` – Note editor (create/edit) with AI toolbar
  - `app/api/notes/*` – CRUD API endpoints for notes
  - `app/api/ai/route.ts` – AI endpoints (summarize / improve / tags)

- `components/` – UI components (NoteCard, NoteEditor, AIToolbar, etc.)
- `hooks/use-notes.ts` – React Query hooks for note operations
- `lib/` – shared logic
  - `lib/auth-context.tsx` – Supabase auth provider + hook
  - `lib/theme-context.tsx` – Light/dark theme provider
  - `lib/groq.ts` – AI integration (GROQ/OpenAI)
  - `lib/validation.ts` – Zod schemas for request validation

- `prisma/schema.prisma` – Database schema

1. **Sign up** using email + password (or sign in if you already have an account)

3. Use the **AI toolbar** in the editor to:
   - Summarize your note
   - Improve grammar and clarity
   - Generate tags automatically

5. Edit or delete notes from the UI

## 🧪 AI Integration (GROQ/OpenAI)

The AI features are backed by the `GROQ_API_KEY` environment variable.

The following actions are supported via `/api/ai`:

- `summarize` – returns a short summary of note content
- `improve` – rewrites note content for grammar and clarity
- `tags` – returns 3–5 comma-separated tags (stored on the note)

> ⚠️ If `GROQ_API_KEY` is missing or invalid, AI requests will fail with an error.
